## 🐛 发现的问题及解决

| 问题 | 根因 | 解决 |
|------|------|------|
| 录屏点停止没反应 | FFmpeg 崩溃后 `recordingProcess` 未清空，close 已触发不再触发 | 异步启动+存活检测 |
| 生成多个损坏录屏文件 | 双击按钮启动多个 FFmpeg | recordingLock 防连点 |
| 删除/重命名弹窗不显示 | `showModal` 参数含 `#`，拼成 `##id` | getElementById |
| 全部按钮无反应 | loading 函数在 `$` 定义前调用 | 移动代码顺序 |

### 录屏文件损坏（moov atom 缺失）— 关键修复
- [x] **根因**: `stdio: ['ignore', ...]` 导致 stdin 不可用，`proc.stdin.write('q')` 静默失败
- [x] FFmpeg 被 SIGTERM 强杀，moov atom 永远无法写入 → 所有录屏文件损坏
- [x] **修复**: 改为 `stdio: ['pipe', 'pipe', 'pipe']`，FFmpeg 收到 'q' 后优雅退出并写完 moov
- [x] 验证: 命令行测试 Duration 正常、moov 无报错、文件可播放
